Co je Právní asistent ČR?

Právní asistent ČR je agent pro M365 Copilot, který vám umožní vyhledávat v plných textech českých zákonů a právních předpisů. Místo procházení e-Sbírky se stačí zeptat přirozeným jazykem.

Agent využívá hybridní AI vyhledávání — kombinaci klíčových slov, vektorových embeddingů a sémantického přehodnocení — takže najde relevantní ustanovení i když použijete jiná slova než jsou v zákoně.

Agent čerpá data z jednoho zdroje:

  • e-Sbírka zákonů a mezinárodních smluv (www.e-sbirka.cz) — otevřená data Ministerstva vnitra ČR, 46 376 právních předpisů od roku 1918 do současnosti, rozčleněných na 112 753 prohledávatelných fragmentů
Agent si nevymýšlí. Všechna data pocházejí z otevřených dat e-Sbírky. Pokud informaci nenajde, řekne vám to — místo toho, aby si ji vymyslel. Vždy cituje doslovné znění zákona.

Jak začít

  1. Otevřete M365 Copilot na webu m365.cloud.microsoft/chat
  2. V levém panelu najděte agenta Právní asistent ČR — kliknutím na něj přepnete konverzaci přímo na agenta
  3. Alternativně můžete do pole pro zprávy napsat @Právní asistent ČR a za zmínkou zadat svůj dotaz
  4. Agent zavolá příslušné API a vrátí vám doslovné znění zákona s citací a odkazem na e-Sbírku
Důležité: Agent je dostupný výhradně v prostředí M365 Copilot. Najdete ho v levém panelu Copilotu nebo ho zavolejte přes @Právní asistent ČR. Pokud napíšete dotaz do obecného Copilotu bez zmínky agenta, nedostanete data z e-Sbírky.

Tipy pro lepší výsledky

Tip 1: Použijte citaci zákona. Pokud znáte číslo zákona, uveďte ho — agent najde přesný paragraf rychleji. Např. „§109 zákoníku práce (262/2006 Sb.)".
Tip 2: Buďte konkrétní. Místo „zákon o práci" zkuste „výpovědní doba pracovního poměru" — agent najde přesnější výsledky.
Tip 3: Používejte filtry. Můžete omezit vyhledávání na typ předpisu (zákon, vyhláška), rok nebo účinnost k datu.
Tip 4: Navazující dotazy. Po vyhledání můžete pokračovat: „Zobraz křížové odkazy na tento zákon" nebo „Kdy nabyl tento zákon účinnosti?"

Jak agent funguje

Agent využívá hybridní AI technologii postavenou na Azure AI Search:

  1. Klíčová slova — klasické textové vyhledávání v 112 753 fragmentech zákonů
  2. Vektorové embeddingy — každý fragment je převeden na vektorovou reprezentaci pomocí modelu text-embedding-3-large, což umožňuje najít sémanticky podobné výsledky
  3. Sémantický reranker — výsledky jsou přehodnoceny AI modelem podle relevance k vašemu dotazu

Díky této kombinaci agent najde relevantní ustanovení i když použijete jiná slova než jsou v zákoně — například „propuštění z práce" najde i §52 zákoníku práce o výpovědních důvodech.
